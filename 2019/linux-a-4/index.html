Linux用户管理

Written by Bing with ♥ on November 9, 2019 in linux
  1. 在linux中，一个用户属于一个组。
  2. 创建一个用户后，默认会在/home/username创建一个和用户名相同的目录，该目录就是用户的家目录
  3. 当切换一个用户登录后，默认会切换到用户家目录
  4. 一个用户默认不能进入其他用户目录
  5. 用户权限体现在对文件的操作

添加用户

useradd [options] username
#执行时，如果没有设置option,系统将自动创建一个和username名字相同的组，并将username用户放进改组。

useradd -d [/home/xx] username #给新创建的用户指定家目录

给用户指定或者修改密码

passwd username #如果没有带用户名，则是给当前登录的用户修改密码

删除用户

userdel username #删除用户，但是保留用户家目录

userdel -r username #删除用户，并且删除用户家目录

查询用户信息

id username

#e.g.
[root@48d2298d7bbd /]# id root
uid=0(root) gid=0(root) groups=0(root)
#uid 用户id,唯一
#gid 用户所在组id
#groups 组名

#当用户名不存在时，返回无此用户

切换用户

su - username
#从权限高的用户切换到权限低的用户，不需要密码，反之需要。
#当需要返回原来的用户时，使用exit指令
#如果su - 没有带用户名，则默认切换到root用户

查看当前用户/登录用户

whoami/who am i

用户组

类似于角色，系统可以对共性的多个用户进行统一管理

新增组

groupadd [组名]

#增加用户时直接加上组
useradd -g [groupname] [username]

删除组

groupdel [组名]
#前提:当前组没有用户时，才能删除

修改用户所在组

usermod -g [newgroupname] [username]

用户和组的相关文件

/etc/passwd

用户（user）的配置文件，记录用户的各种信息

每行的含义：用户名:口令:用户标识号:组标识号:注释性描述:主目录:登录Shell

e.g.

[root@48d2298d7bbd /]# cat /etc/passwd
root❌0:0:root:/root:/bin/bash
#root	 用户名
#x 用户的加密密码，加密后密码保存在/etc/shadow文件
#0 第一个0用户id
#0 第二个0用户组id
#/root 用户家目录
#/bin/bash 使用的shell类型

/etc/shadow #口令配置文件

每行的含义：登录名:加密口令:最后一次修改时间:最小时间间隔:最大时间间隔:警告时间:不活动时间:失效时间:保留

/etc/group

每行含义：组名:口令:组标识号:组内用户列表

e.g.

[root@48d2298d7bbd /]# cat /etc/group
root❌0:
