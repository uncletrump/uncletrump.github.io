Linux RPM和YUM

Written by Bing with ♥ on November 24, 2019 in linux

rpm包的管理

一种用于互联网下载包的打包及安装工具，它包含在某些 Linux 分发版中。它生成具有.RPM 扩展名的文件。RPM 是 RedHat Package Manager(RedHat 软件包管理工具)的缩写，类似 windows 的 setup.exe，这一文件格式名称虽然打上了 RedHat 的标志，但理念是通用的。 Linux 的分发版本都有采用(suse,redhat, centos 等等)，可以算是公认的行业标准了。

查看已安装的rpm列表

rpm -qa #已安装的rpm列表
rpm -a 软件包名 #查看软件包是否安装
rpm -qi 软件包名 #查看软件包信息
rpm -ql 软件包名 #查询软件包中的文件安装到哪里
rpm -qf 文件全路径名 #查看某个文件属于哪个rpm包

卸载rpm包

rpm -e 包的名称

如果其它软件包依赖于要卸载的软件包，卸载时会产生错误信息

强制删除：rpm -e –nodeps 软件包名（不推荐）

安装rpm包

rpm -ivh RPM包全路径名称
i=install #安装
v=verbose #提示
h=hash #进度条

YUM

Yum 是一个 Shell 前端软件包管理器。基于 RPM 包管理，能够从指定的服务器自动下载 RPM 包 并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软件包。使用 yum 的前提是可以联网。

查询yum服务器是否有需要安装的软件

yum list | grep xx

安装指定的yum包

yum install xxx
